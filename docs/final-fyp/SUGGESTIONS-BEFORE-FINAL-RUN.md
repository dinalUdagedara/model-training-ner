# Suggestions before the final thesis run

## Is this folder a good idea?

**Yes.** A single **writing hub** with links to template, sample, checklist, and code paths reduces context-switching and inconsistency. Keep it **light**: link files instead of duplicating the Word template or large PDFs.

## Suggestions

### 1. One working thesis file

- **Copy** `[Template FYP -2025_26] Final Thesis .docx` to something like `CrackInt_Final_Thesis_W1998730.docx`.
- Work only in the copy; keep the original template untouched.

### 2. Version control

- If the thesis is **only** on your laptop, email yourself / cloud backup **dated** copies weekly.
- If you use Git: **do not commit** huge `.docx` binaries unless your team agrees — or use **Google Drive / OneDrive** with version history for the Word file.

### 3. Integrity

- Do **not** paste uncited ChatGPT paragraphs into the thesis; use module **GenAI** rules.
- Any **code** snippet in the report: your own or cited.

### 4. Page limit (template rule)

- **Introduction → Conclusion** = **first 100 pages** marked for content.
- References and appendices are **outside** that limit — plan figure/table placement so core chapters stay within policy.

### 5. Sample FYP usage

- Use `**Pansilu_..._FPR.pdf`** for **structure and depth**, not for copying wording or claims.

### 6. Align IPD with current app

- Your **IPD PDF** may describe an older scope. When reusing text, **diff** against `PROJECT` + `THESIS-FACTS-SHEET.md`.

### 7. Examiners’ path

- **Abstract** and **Conclusion** are read first — make objective achievement and **numbers** crystal clear.
- **Appendix:** only supporting material; no crucial argument only in appendix (template warns examiners may skip it).

### 8. Final week checklist

- All figures have captions + short explanatory text.
- FR/NFR table matches MoSCoW and what you demo.
- Harvard references ≥20, ≥80% strong venues.
- Spell-check supervisor name, module code, student ID.
