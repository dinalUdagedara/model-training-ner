Pansilus-style reading: the presentation tier is summarised as routes + server-driven panels (detail in other chapters). Blue (↓) = HTTP requests; Green (↑) = JSON responses — two parallel spines at the tier boundary (not every route drawn). Grey dashed = optional external LLM.

What we borrowed from a detailed thesis-style diagram (e.g. Pansilus)

  • Orthogonal arrows: Blue ↓ and green ↑ are parallel spines at the presentation–logic boundary (no horizontal buses). They use slightly offset x positions (~598–610) so the two directions do not coincide.
  • Presentation tier (compact): Next.js app shell with one routes & screens band (areas listed inline) and one server-driven output strip — detail screens appear in other figures / chapters.
  • Bidirectional boundary: Blue ↓ and Green ↑ each use one spine through the presentation tier; same HTTP semantics as a multi-arrow diagram with less clutter.
  • Rich logic tier: a central API hub (orange/black) and named pipelines (extraction, both NER paths, session, feedback) plus adapters before the data tier.
  • Data tier + arrows: ↓ for writes/queries initiated from logic; ↑ for data loaded back into the app stack (mirroring loader-style diagrams).
