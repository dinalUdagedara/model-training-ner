Design notes

  • Why this layout: Runtime flows in lane A (top). Data stores sit in a middle band with space between PostgreSQL and S3 so vertical connectors R4 and R5 use left vs right channels and do not cross each other. Training sits in lane B (bottom); R8 rises in a single vertical to the model hub so it does not pass through the Model box horizontally.
  • PostgreSQL vs model files: Checkpoints are files under Model management / S3; PostgreSQL holds application entities (users, sessions, etc.).
