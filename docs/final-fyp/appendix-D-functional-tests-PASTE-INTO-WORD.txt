D.1 Introduction

This appendix documents black-box functional tests derived from the Software Requirements Specification (Chapter 04), executed against the CrackInt prototype (FastAPI backend, Next.js frontend). Tests verify integration between the browser, REST API, database, NER checkpoints (Appendix G), and optional LLM features.

Testing was performed manually in the same environment used for demonstration (local or deployment URL, version and date: [e.g. 2025-03-20]). Where an external provider is required (OpenAI for session question generation and answer evaluation), outcomes are recorded as Pass, Partial (degraded but documented behaviour, for example HTTP 503 when the key is missing), or Fail. D.2 Traceability and method

Each test maps to one or more functional requirement (FR) identifiers from the SRS. Preconditions state what must be true before the test (for example registered user, valid JWT). Steps summarise UI or API actions; expected states the acceptance criterion; actual records what was observed (HTTP code, key JSON fields, or UI state). Evidence (Swagger responses, screenshots) may be placed in Section D.3 if the module requires it.
