How to read: (1) Every class lists id, created_at, and updated_at (implemented in code via SQLModel mixins). (2) Blue links R1–R6 are foreign keys: the open arrowhead sits on the class that holds the FK column (the “many” side of 1:*). (3) Small 1 / * labels match the catalogue table. (4) Long associations (R2–R4) use separate horizontal bands just below the class row so lines do not cross the title or stack on one path.
