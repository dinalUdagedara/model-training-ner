# Viva PPT Rehearsal Checklist (One Page)

Use this on the day before and on viva day.

---

## 1) 20-Minute Timing Control

- [ ] Total run is exactly 20:00 (not under, not over).
- [ ] Slide-level timing follows `VIVA-PPT-FULL-PACK-20MIN.md`.
- [ ] Demo fits inside 3:30.
- [ ] I practiced one run with stopwatch and no pauses.
- [ ] I practiced one run with interruptions (to recover quickly).

---

## 2) Non-Negotiable Slide Content Check

- [ ] Problem is clearly explained with LR context.
- [ ] Research gap is very clear (strongest section).
- [ ] Aim and objectives are aligned to implementation/evidence.
- [ ] Architecture/design is shown with diagram.
- [ ] Implementation and tech stack are clearly shown.
- [ ] AI/ML details include dataset, preprocessing, training, experiments.
- [ ] Testing and evaluation evidence are clearly presented.
- [ ] Critical evaluation outcome is included.
- [ ] Contribution to body of knowledge and novelty are clearly stated.
- [ ] Limitations and future work are explicit.
- [ ] Conclusion is concise and evidence-backed.
- [ ] Existing skills and new skills are clearly listed.
- [ ] Final summary slide is clear and strong.

---

## 3) Numbers and Evidence Memory Check

- [ ] Resume NER micro-F1: `0.83`
- [ ] Job-poster NER micro-F1: `~0.85`
- [ ] Resume dataset frozen run: `4738` (split `3790/473/475`)
- [ ] Job dataset frozen run: `6327` (split `5061/632/634`)
- [ ] I can explain achieved vs partial objectives clearly.
- [ ] I can mention at least 3 testing limitations honestly.

---

## 4) Demo Readiness Check

- [ ] Login works.
- [ ] Resume extraction demo sample ready.
- [ ] Job extraction demo sample ready.
- [ ] Session question/answer turn ready.
- [ ] Readiness summary screen ready.
- [ ] Backup screenshots prepared in correct order.
- [ ] Backup short video available.
- [ ] If demo fails, I can continue calmly with evidence.

---

## 5) Defense Readiness Check

- [ ] "How is this different from ChatGPT?" answer ready.
- [ ] "Are you claiming global uniqueness?" safe answer ready.
- [ ] "Why this model?" answer ready.
- [ ] "Why this methodology?" answer ready.
- [ ] "Why trust your evaluation?" answer ready.
- [ ] "Biggest limitation?" answer ready.
- [ ] "If you had more time, what first?" answer ready.

Use this answer structure every time:
- Position -> Reason -> Evidence -> Limitation -> Next step.

---

## 6) Delivery Quality Check

- [ ] Speak in simple, clear English.
- [ ] No overclaiming.
- [ ] No defensive tone when discussing limitations.
- [ ] Keep transitions smooth between slides.
- [ ] Keep eye contact and stable pace.
- [ ] Close with 3 key takeaways.

---

## 7) Final Device and File Check

- [ ] PPT file opens correctly.
- [ ] Report PDF is locally available.
- [ ] Internet/power backup plan prepared.
- [ ] Charger and adapter ready.
- [ ] Demo environment pre-opened before session.
- [ ] Backup files are in one folder.

---

## 20-Second Opening Script

"Good [morning/afternoon]. I am presenting CrackInt, my AI-driven interview preparation platform. I will explain the problem and gap, then design and implementation, then demo and evaluation results, and finally contribution, limitations, and conclusion."

## 20-Second Closing Script

"In summary, I delivered a working integrated platform with measurable evidence, clear critical evaluation, and realistic future improvements. Thank you."
